The Making of UCSD Alert Depot

Building an open platform for secure, accessible safety intelligence

Our vision is to create an intuitive system that empowers communities with accessible safety information through advanced geospatial visualization. By developing flexible APIs and intelligent location parsing, we're building infrastructure that any organization can use to securely plot and visualize safety data‚Äîfrom campus alerts to city-wide incident reports.

We aim to bridge the gap between complex safety data and public awareness, making critical information more actionable and accessible to everyone.

Our Core Mission

Our goal is to develop intelligent AI agents for geocoding location descriptions with large language models and create interactive dashboards to find and visualize alerts across San Diego, starting with the UCSD campus.

AI-Powered Geocoding

Creating specialized agents that leverage LLMs to understand complex location descriptions and translate them into precise coordinates, even for campus-specific references that traditional geocoding services miss.

Interactive Safety Dashboards

Developing intuitive visualization tools that transform raw alert data into actionable insights, helping users understand safety patterns and make informed decisions.

Problem Statement

UCSD campus alerts often reference location names that global mapping services don't recognize ‚Äì such as "Price Center," "Ridgewalk," or residence hall names. Standard geocoding fails to translate these campus-specific references into precise coordinates.

Our goal was to create a system that accurately maps alerts when they mention locations like "Lot P103 near Revelle" or "BCB Caf√© at Warren Mall" ‚Äì providing a visual interface that helps students stay informed about campus safety.

The Problem

Traditional geocoding services like Google Maps cannot accurately locate many campus-specific buildings, landmarks, and areas.

Process Overview

Given our problem statement and known geocoding challenges, we began experimenting and developing our product.

Here is an outline of our development process:

  1. Create script to scrape crime data periodically from UCSD Police Crime Logs
  2. Process and clean text data using simple NLP techniques
  3. Explore geocoding options: Geopandas + LLMs to locate and identify specific campus locations
  4. Spatial density analysis to visualize data distribution
  5. Use most accurate geocoding to create our interactive prototype
  6. Future directions: refine accuracy in geocoding and NLP and explore broader applications

Real-Time Data Processing

To provide timely information, we created a pipeline that automatically scrapes safety alerts from the UCSD Police Department website, processes the text to extract key details (location, incident type, date/time), and geocodes the locations using our custom database.

AI-Enhanced Geocoding

For ambiguous location descriptions, we implemented NLP and language models to match unclear references to known locations in our database.

LLM Geocoding Comparison

We compared the performance of various Large Language Models for geocoding UCSD-specific locations. Different models showed varying levels of effectiveness for campus landmark recognition.

Our tests revealed that while all LLMs had decent accuracy for well-known locations like "Geisel Library," specialized models like Mistral Large showed superior performance for ambiguous campus-specific references.

Traditional geocoding services like ArcGIS struggled most with campus-specific vernacular such as building nicknames and informal areas, confirming the need for our specialized solution.

LLM-Powered Geocoding with Mistral

A key innovation in our system is the use of Large Language Models (LLMs) for intelligent geocoding. We implemented Mistral AI to improve location processing:


# Using Mistral AI to improve geocoding
from mistralai import Mistral

def geocode_unknown_locations_with_mistral(unknown_locations):
    """
    Use Mistral AI to geocode unknown location addresses to coordinates.
    Returns: Dictionary mapping each location to its coordinates and address
    """
    client = Mistral(api_key=API_KEY)
    
    # Build a detailed prompt for the LLM to understand campus context
    prompt = build_geocoding_prompt(unknown_locations)
    
    response = client.chat.complete(
        model="mistral-large-latest",
        messages=[{"role": "user", "content": prompt}]
    )
    
    # Extract and process the structured JSON response
    result_text = response.choices[0].message.content
    geocoded_results = json.loads(extract_json_from_response(result_text))
    
    locations_dict = {}
    for result in geocoded_results:
        location = result["location"]
        name = result["name"]
        address = result["address"]
        lat = float(result["lat"])
        lng = float(result["lng"])
        
        locations_dict[location] = {
            "name": name, 
            "address": address,
            "lat": lat, 
            "lng": lng
        }
    
    return locations_dict

By structuring detailed prompts to the Mistral model, we achieved highly accurate geocoding that understands complex location descriptions like "near the main entrance of Geisel Library" or "Parking Lot P103 northwest corner."

API Development for Interoperability

We designed flexible REST APIs to make our system's capabilities accessible to other applications and organizations. This allows our geocoding engine and visualization tools to integrate seamlessly with existing safety systems.


# FastAPI endpoints for external access to our geocoding and alert data
@app.get("/api/geocode", response_class=JSONResponse)
async def geocode_location(
    location_text: str,
    format: Optional[str] = Query("json", regex="^(json|geojson)$")
):
    """
    Geocode a location description using our UCSD-specific database.
    Returns coordinates and formatted address.
    
    Query parameters:
    - location_text: The text description to geocode
    - format: Return format (json or geojson)
    """
    try:
        # Process through our geocoding pipeline
        lat, lng, address = geocode_campus_location(location_text)
        
        if not lat or not lng:
            # Fall back to LLM if traditional lookup fails
            result = await geocode_with_llm(location_text)
            if result:
                lat, lng, address = result
        
        if format == "geojson":
            return {
                "type": "Feature",
                "geometry": {
                    "type": "Point",
                    "coordinates": [lng, lat]
                },
                "properties": {
                    "address": address,
                    "query": location_text
                }
            }
        else:
            return {
                "lat": lat,
                "lng": lng,
                "address": address,
                "query": location_text
            }
    except Exception as e:
        return JSONResponse(
            {"error": f"Geocoding failed: {str(e)}"},
            status_code=500
        )
